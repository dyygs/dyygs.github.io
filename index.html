Intent是什么？

An intent is an abstract description of an operation to be performed. It
can be used with {@link Context#startActivity(Intent) startActivity} to
launch an {@link android.app.Activity}, {@link android.content.Context#sendBroadcast(Intent) broadcastIntent} to
send it to any interested {@link BroadcastReceiver BroadcastReceiver} components, and {@link android.content.Context#startService} or {@link android.content.Context#bindService} to communicate with a background {@link android.app.Service}.

An Intent provides a facility for performing late runtime binding between the code in different applications. Its most significant use is in the launching of activities, where it can be thought of as the glue between activities. It is basically a passive data structure holding an abstract description of an action to be performed.

照例引用官方class annotation，Intent 作为将要执行的操作的一个抽象描述，来表达我们的一些意图，可以使用在applications，也可以使用在 Activity Service BroadcastReceiver组件上，来完成不同applications，或者不同组件之间的交互和通信。

Intent如何使用（着重在activity）？

Intent属性：

  • component：目的组件
  • action：用来表现意图的行动
  • category：用来表现动作的类别
  • data：表示与动作要操纵的数据
  • type：对于data范例的描写
  • extras：扩展信息
  • Flags：期望这个意图的运行模式

Intent分为显式intent 和 隐式intent，区别在于隐式intent不指定component，而是让系统根据action,category等属性去自主决定启动哪个

component 直接指明目的组件

上述写法的关键点都一样,都是声明package的name和目标组件class的name：

Action 用在Manifest文件中，在intent-filter指定intent能接受的动作
Category 指明动作类别 和 Action 一起使用

上述代码中 自定义action，必须配合默认的category，也就是说action和category必须都匹配上，才认为是匹配成功，如果有多个匹配成功，则系统会弹出给用户选择要运行的组件

data 指定要操作的数据，常用在打开系统浏览器浏览网页
type 对于data类型的描述，常用于打开资源文件
extras 扩展信息
flags 启动模式（涉及内容复杂，单独在下一篇学习）

拨打电话，发送短信等功能参考intent解析

Looper是什么？

Class used to run a message loop for a thread. Threads by default do
not have a message loop associated with them; to create one, call
{@link #prepare} in the thread that is to run the loop, and then
{@link #loop} to have it process messages until the loop is stopped.

引用源码中的annotation,Looper能为一个线程循环一个消息队列，默认情况下，线程不开启Looper（UI线程是开启的）,所以我们需要通过调用Looper.prepare()来初始化Looper，然后调用Looper.loop()使其循环开始，直到调用Looper.quit();

Looper怎么使用？

  1. 新建Looper线程
  1. 在中，调用LooperThread的 mHandler发送对象

Looper.prepare()做了什么？

可以看到prepare的过程就是初始化Looper，并且和当前线程绑定的一个过程

Looper.loop()做了什么？

除去冗余代码，主要是通过一个for循环，来不断的对msg进行分发

结合上篇的handler，来看一下dispatchMessage(msg)

  1. 如果Message自带了Callback，则交给Message的Callback处理；
  2. 如果Handler了设置了Callback，则交给Handler的Callback处理；
  3. 如果两者都没有，则调用handleMessage方法处理。

到这一步，就把位于threadOfMsg线程的msg发送到了looperThread，完成了线程间的通信

UITextField输入字符限制(中英文混编)

发表于 2016-10-14  |  2016-11-14   |   分类于 ios学习笔记   |     |   阅读次数
题记：

在ios的开发中，总会遇到要对输入的字符长度进行限制，目前网上大多提供的方法都是单纯的对 [textField.text length] 的简单判断，显然这种方法在面对中英文混合的时候并不适用，因为我们想要限制输入20个字符的时候，对应的中文字符应该是10个。

处理步骤：
  • 添加监听
  • 统计已输入字符的length(中文个数 * 2 + 英文字符个数)
  • 统计在字符长度限制下的maxLength
  • 对于超过长度的字符串进行截取
示例代码：
- (void)textFieldDidChange:(UITextField *) textField
{
    if (textField == self.nicknameTextField) {
        NSInteger lengthOfChiness = 0, lengthOfEn = 0;
        NSString *toBeString = textField.text;
        NSInteger maxLength = 0;
        for(int i=0; i< [toBeString length];i++){
            unichar a = [toBeString characterAtIndex:i];
            //判断是否是中文字符
            if( a > 0x4e00 && a < 0x9fff)
            {
                //统计中文个数
                lengthOfChiness += 1;
            } else {
                //统计英文个数
                lengthOfEn +=1;
            }
            if ((lengthOfEn + lengthOfChiness * 2) <= 20) {
                //长度限制下的最大字符个数
                maxLength = lengthOfEn + lengthOfChiness;
            } else {
                textField.text = [textField.text substringToIndex:maxLength];
                [[iToast makeText:@"超过字符限制"] show];
            }
        }
    }
}

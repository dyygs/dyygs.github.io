Git学习笔记

发表于 2017-02-24  |  2017-02-27   |   分类于 版本控制   |     |   阅读次数

Git是什么？

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

Git is easy to learn and has a tiny footprint with lightning fast performance. It outclasses SCM tools like Subversion, CVS, Perforce, and ClearCase with features like cheap local branching, convenient staging areas, and multiple workflows.

照例附上官方简介：Git作为一个免费且开源的分布式版本管理系统，能够快速且高效的完成从小项目到大项目需要的一切。相较于其他的版本管理工具，Git轻量级且更快速，更廉价的本地分支，易用的暂存区和多个工作流。

Git如何用？

1. 创建本地仓库

git init ：把当前目录初始化成一个git管理的仓库

2. 添加文件到stageArea

git add : 把当前目录的文件添加进stageArea（暂存区）

3. 添加文件到版本库

git commit -m “commit msg” ：把暂存区的修改，提交到版本库

4. 查看仓库状态

此时，我们修改test.md的内容

git status : 查看当前仓库状态

5. 查看更改内容

git diff : 查看修改之后的不同

5. 撤销修改（尚未添加到暂存区）

git checkout – filename

此时还原到上次git commit 时的状态

6. 撤销修改（添加到暂存区后，再次修改）

git checkout – filename

此时还原到git add 时的状态

7. 回退版本

需要回退到某个历史版本

git log – filename

git log 显示从最近到最远的提交日志

在Git中，用HEAD表示当前版本，上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100

git reset –hard HEAD 回退到上个版本

git reflog 显示历史命令

8. 删除文件

git rm filename

9. 添加远程仓库

git remote add origin “github.address”

10. 克隆远程仓库,并且提交本地文件到远程仓库

git clone “github.address”

git push -u origin master 用于首次推送，-u 可以把本地master和origin master 关联起来

git push origin master 除了首次之外，其他的便可以这样推送

11. 分支

查看分支：git branch

创建分支：git branch “branchName”

切换分支：git checkout “branchName”

创建+切换分支：git checkout -b “branchName”

合并某分支到当前分支：git merge “branchName”

删除分支：git branch -d “branchName”

12. 暂存工作区

git stash 暂存工作去内容
git stash list 暂存历史
git stash apply 应用暂存内容
git stash drop 丢弃暂存内容
git stash pop 等同于 apply + drop
